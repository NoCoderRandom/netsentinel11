# Security

## Scope and policy

NetSentinel11 is local-first and defensive. It is intended for:
- Networks owned by the user, or explicitly authorized for assessment
- Authorized troubleshooting, auditing, and inventory tasks

## Reporting security issues

If you find a security issue in this repository:

1. Do not publish the report publicly.
2. Use GitHub's private vulnerability reporting feature when available.
3. If private vulnerability reporting is unavailable, contact the maintainer privately before opening a public issue.
4. Include OS version, app version, reproduction details, and whether the issue requires local network access.

## Safety defaults

This project never ships with traffic-disruption behavior by default.
Exploit payloads, credential attacks, ARP spoofing, deauth, MITM, and similar tactics are forbidden.

## Public issue safety

Do not include passwords, public IP addresses, router exports, private scan logs, or personally identifying information in public issues.
