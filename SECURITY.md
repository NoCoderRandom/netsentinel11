# Security

## Scope and policy

NetSentinel11 is local-first and defensive. It is intended for:
- Networks owned by the user, or explicitly authorized for assessment
- Authorized troubleshooting, auditing, and inventory tasks

## Reporting security issues

If you find a security issue in this repository:

1. Do not publish the report publicly.
2. Open an issue marked `security` with steps to reproduce.
3. Include OS version, app version, and reproduction details.

## Safety defaults

This project never ships with traffic-disruption behavior by default.
Exploit payloads, credential attacks, ARP spoofing, deauth, MITM, and similar tactics are forbidden.
